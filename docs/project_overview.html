01 — Project Overview

ロボットに「見て・考えて・動く」AIを搭載する

⚠️
Problem
従来ロボットの限界
  • 固定ルールベースの動作制御
  • 環境変化への適応が困難
  • カメラ映像の意味理解が不可
  • 再学習・更新コストが高い
🤖
Solution
Robotics AI Platform
  • ROS 2 でリアルタイム制御統合
  • PyTorch による物体認識・推論
  • OpenCV で高速画像処理
  • Docker で環境を完全再現
✅
Outcome
期待するアウトカム
  • モデル差し替えが容易な設計
  • 30Hz リアルタイム推論対応
  • CI/CD で品質を継続担保
  • Gazebo シミュレーション対応
Key Takeaway ROS 2 の制御フレームワークと深層学習推論エンジンを統合し、モデル交換可能な拡張性の高いロボット AI プラットフォームを構築する

03 — Development & CI/CD

Docker 1コマンドで完全な開発環境を即時再現

01
リポジトリをクローン
git clone https://github.com/
shotoku0063/robotics-ai.git
cd robotics-ai
02
Docker 環境を起動
cd docker
docker-compose up -d
docker-compose exec \
robotics-ai bash
03
ROS 2 ワークスペースをビルド
source /opt/ros/humble/setup.bash
cd ros2_ws
colcon build --symlink-install
source install/setup.bash
Gazebo シミュレーター を使用する場合： docker-compose --profile simulation up
Key Takeaway Docker により「自分のマシンでは動く」問題を根絶 ―― ROS 2 / PyTorch / OpenCV の複雑な依存関係をコンテナに封じ込める

Lint

PR / main push
→
Job 2
Unit Tests
  • pytest 8 テスト実行

  • カバレッジレポート生成
PR / main push
→
Job 3
Docker Build
  • ROS 2 フルビルド検証
  • colcon build 確認
  • Buildx キャッシュ有効
main push のみ
Key Takeaway PR では Lint + Test のみ（高速）、main マージ時に Docker フルビルドを実行することで速度と網羅性を両立

How to Run

主要コマンドリファレンス

🧪 ローカルテスト実行
$ pip install pytest pytest-cov
$ pytest tests/ -v
$ pytest tests/ --cov=ros2_ws/src/robotics_ai/robotics_ai
🐳 Docker 起動
$ cd docker
$ docker-compose up -d
$ docker-compose exec robotics-ai bash
🤖 ROS 2 ノード起動
$ source ros2_ws/install/setup.bash
$ ros2 run robotics_ai perception_node
📈 モデル学習
$ python scripts/train.py \
  --data data/processed \
  --epochs 100 \
  --output models/
