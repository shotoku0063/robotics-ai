#1
apt install で停止
ros-humble-ur-simulation-gazebo が apt バイナリ配布されていないことを発見。UR 公式は GitHub ソースのみ提供。
CI 56秒 · apt 失敗
8e0babb
Dockerfile 内 git clone
apt 通過したが、bind mount が /workspace を上書きしてイメージ内のソースが隠れる。「launch がパッケージを見つけられない」エラー。
CI 7m26s · ros2 launch 失敗
b27afa0
CI / record_demo.sh 側で vendor clone
パッケージは見つかった。しかし sim_demo の libexec ディレクトリが install/sim_demo/lib/sim_demo に作られていない。
CI 8m6s · libexec dir 欠落
40465eb
--symlink-install を外して install ツリーを可視化
entry-point スクリプトが /bin/ に入っていて /lib/sim_demo/ に無いと判明。ament_python 標準の setup.cfg 不在が原因。
CI 7m54s · 同じ libexec 失敗だが原因特定
76f22b7
setup.cfg で install_scripts を $base/lib/sim_demo に
libexec dir が出来て launch がノードを発見、pick_and_place が起動。だが scaled_joint_trajectory_controller に接続しようとして失敗（sim では prefix なし）。
CI 9m9s · action server 未接続
4bf1a6d
controller トピック名 + gazebo 重複削除 + 黒フレーム fallback
Pick & Place 21キーフレームが全て実行された。但しカメラがまだ世界に存在せず、video_recorder はフォールバックで黒画面 MP4 を出力。
CI 9m21s · MP4 46KB
0897afb
カメラ SDF を TimerAction + spawn_entity で投入
カメラは spawn 成功したが、SDF 内 remapping が効かず /overhead_cam/image_raw に publish。/camera/image_raw を購読する video_recorder は受信できず黒画面のまま。
CI 6m51s · MP4 61KB
c66b211
camera_name=camera で直接 /camera/image_raw に publish
video_recorder が 2,250 frames（75秒 × 30fps）を取得、UR5e の動きが映る MP4 5.2MB が完成。E2E パイプライン全通過。
CI 6m34s · MP4 5.2MB (実映像)
読み方 ―― 各イテレーションは前段の失敗の根本原因をひとつだけ潰している（apt → Dockerfile → workspace → install layout → controller name → camera topic）。 並走してデバッグ情報を残すコミット (#4) を1本入れたことで、原因特定の速度が上がった。
